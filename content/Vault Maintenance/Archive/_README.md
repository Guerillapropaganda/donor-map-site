# Vault Maintenance/

Infrastructure and process documents that govern how the vault operates.

## Key Files

| File | Purpose |
|------|---------|
| **Session Timeline.md** | Recent session history (last 2 weeks), current priorities |
| **_SESSION_HISTORY_ARCHIVE.md** | Archived sessions (older than 2 weeks) |
| **Publish Roadmap - The Donor Map Database.md** | Phased execution plan, bottleneck map, milestone targets |
| **Research Methodology and Data Sources.md** | Research standards, source format, analytical framework |

## Rules

- Session Timeline must be updated at the END of every session
- When Session Timeline exceeds ~2000 lines, archive older sessions to _SESSION_HISTORY_ARCHIVE.md
- The Publish Roadmap is the strategic document — check it for current phase before starting work
