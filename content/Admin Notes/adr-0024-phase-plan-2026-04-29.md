## Five-phase plan

Each phase is sized for one session unless flagged otherwise.

### Phase 1 — Complete plumbing (`paths`, `subgraph`, `timeline`)

**Why first:** thesis layer functions compose from plumbing. `influencePipelines`
sits on `paths`. `subgraph` powers the public graph view. `timeline` is the
profile-page activity feed.

**Scope:**
- `Graph.paths(from, to, opts)` — BFS up to maxHops with per-edge filters,
  ranked by sum-of-edge-weights desc. Edge weight = amount when present,
  else confidence.
- `Graph.subgraph(seedIds, maxHops, opts)` — flood-fill from seeds, return
  `{ nodes: Node[], edges: Edge[] }`. Bounded result (cap nodes at e.g.
  5000 with `truncated: bool`).
- `Graph.timeline(seed, opts)` — chronological edge list. Sort by
  `first_seen` desc. Filter by `edge_types`, `cycle`, `status`.
- 3 unit tests in `__tests__/graph.test.ts`.
- No migration of consumers in this phase — just land the API.

**Estimate:** ~$8-12 Opus, single session.

**Done in:** 2026-04-29 (Phase 1 of this plan).

### Phase 2 — First thesis query: `donorContradictions`

**Why:** simplest of the 8 thesis queries (no policy data, no voting record
needed — just same-donor → opposing politicians). Establishes the pattern
for future thesis queries.

**Scope:**
- `Graph.donorContradictions(donorId, opts?)` — find politicians the donor
  funds who have political-opposition edges between each other (directly or
  via shared opposes target).
- Return shape: `{ donor: Node, pairs: Array<{ a: Node, b: Node, total_to_a, total_to_b, opposition_basis: Edge[] }> }`.
- Deduplicate undirected pairs; rank by minimum total dollars (the
  story-potential heuristic — both sides being funded substantially).
- Unit tests covering the Bowman+UDP / Cori Bush+UDP shape, fixture-backed.
- Wire to `/audit-claude-decisions` is NOT this phase — that's Phase 4.

**Estimate:** ~$8-12 Opus, single session.

### Phase 3 — Marquee thesis query: `influenceMap`

**Why:** the central piece of the donor-map thesis ("money flows shape
ideology and policy"). Highest reader value once shipped.

**Scope:**
- Extend loader to read `data/policies.jsonl` (and any vote-record store —
  audit during the session).
- Add `policy` and `vote-on-policy` to NodeType / EdgeType.
- `Graph.influenceMap(politicianId)` — top donors with class tags +
  politician's votes on related policies + alignment score per cluster.
- Score formula: define + document. Likely cosine similarity between
  donor-cluster's `ideological_function` set and politician's vote pattern
  on policies tagged with overlapping `policy_stakes`.
- Tests covering the AOC-shape (top labor donors + progressive vote
  alignment) and a contrasting Manchin-shape if data supports.

**Estimate:** ~$15-25 Opus, possibly two sessions if loader audit reveals
schema gaps.

### Phase 4 — Migrate `/profile` to use thesis queries

**Why:** highest-traffic ops surface; biggest data-integrity payoff.

**Scope per ADR-0024 §migration:**
- Identify which `/profile` panels still read frontmatter caches or
  per-page loaders.
- Cut over to `Graph.aggregate` + `Graph.influenceMap` + `Graph.classProfile`.
- Run shadow mode for one week — every old read also calls librarian and
  logs diff. Walk diffs with David before deleting old paths.
- Same shadow-mode pattern for `/relationships`, `/money-trail` afterward.

**Estimate:** ~$10-15 Opus per surface, ~3 sessions total.

### Phase 5 — UI consolidation: `/explore`

Per ADR-0024 §migration step 6: merge `/relationships + /money-trail +
/capitol-trades + /connections + /contradictions` into one `/explore`.
Mechanical UI work once backends share the librarian. Deferred until Phase 4
completes for all those surfaces.

**Estimate:** ~$15-20 Opus, possibly two sessions.

## Open questions deferred to specific phases

- **Phase 3:** how to compute alignment score (ADR-0024 §open-questions Q2).
  Defer until the schema audit reveals what voting-record signal is
  available.
- **Phase 4:** what to do when shadow-mode diffs are large. Defer to the
  migration session — likely most diffs are "old was wrong, new is right"
  but a few may surface schema bugs.
- **All phases:** structured data vs rendered text from thesis queries
  (ADR-0024 §open-questions Q1). Default to structured; let surfaces
  decide presentation.

## Sequencing notes

1. **Phase 1 unblocks everything** but is small enough to ship today.
2. **Phase 2 is independent of Phase 3** — donorContradictions only needs
   monetary + opposition edges, both already loaded.
3. **Phase 3 has the largest variance** ($15-25 to maybe $30+) — depends
   on whether policies.jsonl is well-shaped or needs cleanup.
4. **Phase 4 is the most David-attention-heavy** — shadow-mode diff review
   needs editorial judgment per surface.
5. **Phase 5 is pure UI work** — fast once 4 is done.

Total estimate: 5-7 sessions, ~$80-130 Opus end-to-end. Matches the
ADR-0024 estimate ("4-6 sessions" plus UI).
