# Profile Template

Every verified profile on thedonormap.org follows the same 9-section structure, in the same order. A reader who has read one profile knows where to find any fact on the next.

This is enforced by `scripts/profile-template-validator.cjs` (pre-commit sentinel) for any profile with `content-readiness: verified`.

## The 9 sections

1. **Summary Infobox** — auto-generated from frontmatter + edges
2. **Who They Are** — editorial, 2-3 paragraph lede
3. **Class Analysis** — editorial, 300-500 words. Frames the rest of the profile.
4. **The Money** — auto from edges, with editorial framing
5. **Key Votes / Executive Actions / Politicians Funded / Contracts + Lobbying** — type-specific auto section
6. **The Contradictions** — auto from contradiction-scanner + editorial framing
7. **Timeline** — auto from events + FEC cycles + editorial annotations
8. **Related Figures** — auto from `data/relationships.jsonl` (not frontmatter)
9. **Sources** — auto from source registry

**Section 5 type-specific rules:**
- Federal legislators (House/Senate): "Key Votes"
- **Presidents / Cabinet** (`chamber: "Presidential"` or `"Cabinet"`): **"Executive Actions"** — EOs, proclamations, agency directives, Cabinet appointments, not votes
- State/local politicians: "Key Votes"
- Individual donors: "Politicians Funded"
- PACs: "Allied Donors + Politicians Funded"
- Corporations: "Contracts + Lobbying"
- Think tanks: "Policy Positions"
- Lobbying firms: "Clients + Issues"

**Why Class Analysis is at Section 3:** it frames everything that follows. The reader sees the analytical lens before the evidence. Every data section afterward (money, votes/actions, timeline) sits inside the class-analytic frame, not separate from it.

Sections marked "auto" are generated by scripts reading canonical stores. Sections marked "editorial" are written by Research Claude and signed off by David.

## Section 1: Summary Infobox

**Purpose:** every big fact in the first viewport. No scrolling required to know who this is and why they matter.

**Fields rendered by `SummaryInfobox.tsx`:**
- Name + pronunciation guide (if unusual)
- Type badge (politician / donor / corporation / state-politician / local-politician)
- Party + chamber + state (politicians) OR sector + entity-type (donors/corps)
- Current office + tenure start date (politicians)
- Career total raised / total funded / total contracts
- Top 3 donors / top 3 politicians funded / top 3 contract agencies
- Data freshness: "as of [cycle]" stamp
- Readiness badge (verified / ready / draft / raw)
- 2026 race status (if applicable): incumbent / challenging / retired / ran-lost

**Data source:** frontmatter fields generated from canonical stores via `scripts/build-profile-data-panels.cjs`.

**Placement:** first thing after the title. Visible above the fold on desktop and mobile.

**Custom stats for outlier profiles:** profiles with extraordinary financials that don't fit the standard schema (Trump's Truth Social stake, $TRUMP coin holdings, World Liberty Financial proceeds, etc.) can add a `custom-stats` frontmatter array:

```yaml
custom-stats:
  - label: "Trump Media (DJT) stake"
    value: "~$3B at peak"
    source: "SEC 10-K filings"
  - label: "$TRUMP meme coin holdings"
    value: "800M of 1B tokens"
    source: "On-chain (Solana)"
```

SummaryInfobox renders these in a "Additional Tracked Financials" section. Non-outlier profiles omit the field. Only use for genuinely outlier facts — not substitute for Section 4 (The Money) auto-generated data.

## Profile page tabs

Long profiles are split into tabs so readers don't endless-scroll. Built automatically by `ProfileHeader.tsx` (assigns `data-tab` to each section card) + `ProfileTabs.tsx` (renders the tab nav).

Tab mapping (automatic, based on section heading):

| Tab | Profile Type | Sections included |
|-----|-------------|-------------------|
| Overview | all | Who They Are, Class Analysis |
| The Money | politician, donor, corp | Section 4 (Money) + related supps (donor sectors, policy returns) |
| Key Votes | politician | Section 5 (Key Votes) |
| Executive Actions | presidential / cabinet | Section 5 (Executive Orders) |
| Recipients | donor | Politicians Funded + related supps |
| Analysis | all | Contradictions, Timeline, analytical essay supps |
| Sources | all | Related Figures, Sources, Archived |

On mobile (<800px), tabs collapse to vertical accordion. Reader can expand sections one at a time instead of clicking tabs.

## Section 2: Who They Are

**Purpose:** the lede. Who is this person and why should I care.

**Format:** 2-3 paragraphs of editorial prose. Max 250 words. Must establish:
1. Basic biographical fact (born where, career arc)
2. Current role + institutional power
3. One-sentence answer to "why does this profile exist"

**Voice:** direct, punchy, brutalist. Short sentences. No em dashes (banned). No AI-vocab (delve, moreover, testament to, tapestry, etc). No defamation-prone words outside blockquotes (fraud, corrupt, scheme, bribed — flagged by `self-review-mirror`).

**Example (Trump):**
> Donald John Trump. Born June 14, 1946, in Queens, New York. Real estate heir turned casino bankrupter turned reality TV host turned president turned convicted felon turned president again.
>
> 47th President of the United States. Net worth fluctuates between $2B and $6B, most of it in branding and Truth Social stock. Born into wealth, not self-made, despite the mythology. Fred Trump's real estate empire provided the inheritance, the loans, and the political infrastructure.
>
> This profile exists because Trump is the largest single recipient of political spending in American history and the clearest case study in how the donor class buys policy through a transactional president.

## Section 3: The Money

**Purpose:** show the money visually, not as prose.

**Auto-rendered components:**
- Career total stat block (large number, cycle breakdown underneath)
- Top 10 donors table with: name, amount, cycle(s), relationship type, edge-count
- Sector breakdown donut (where the money comes from)
- Year-over-year bar chart
- Link to full donor list (paginated)

**Data source:** `data/relationships.jsonl` filtered by this entity.

**Editorial slot:** 1 paragraph framing at top explaining what the pattern reveals. Research Claude writes.

**Example framing (Koch Industries):**
> Koch's political spending has three tiers. Direct corporate PAC contributions are the visible layer at $25M per cycle. The dark-money layer — Americans for Prosperity, Stand Together, the network of 501(c)(4)s — is an order of magnitude larger and not required to disclose donors. What's tracked here is the visible layer.

## Section 4: Type-specific auto section

### For politicians: Key Votes + Actions

- Auto list from GovTrack: top 20 floor votes in current cycle, with bill title + outcome + donor alignment
- Auto list from Congress.gov: bills sponsored, committee assignments
- Editorial slot: 1 paragraph identifying the 3-5 votes that reveal the class position

### For donors (individual): Politicians Funded

- Auto table: politician name, total given, cycles covered, party breakdown
- Auto map: by-state heatmap of giving
- Editorial slot: 1 paragraph identifying the cross-party pattern if present

### For donors (PAC): Allied Donors + Politicians Funded

- Auto list of top individual contributors to this PAC
- Auto list of politicians the PAC funds
- Editorial slot: 1 paragraph on the PAC's ideological niche

### For corporations: Contracts + Lobbying

- Auto list: top federal contracts by agency (USASpending data)
- Auto list: lobbying spend by year (LDA data)
- Auto list: revolving-door personnel (hand-curated, grows over time)
- Editorial slot: 1 paragraph connecting contracts + lobbying + politicians funded

## Section 5: Class Analysis

**Purpose:** the editorial lens. Class-analysis framing that distinguishes this site from OpenSecrets.

**Format:** 300-500 words of editorial prose. Written by Research Claude per `content/Vault Rules.md` class-analysis requirements.

**Must anchor on data from Section 3, not invent new facts.** Focus on:
1. What fraction of capital this entity represents (finance, tech, fossil, carceral, defense, pharma, etc.)
2. Ideological function — which policy demands track to the money
3. Cross-party spending if any, and what it reveals
4. Position in the broader capital-to-politician pipeline

**Vocabulary:** locked by ADR-0001 (5-dimension schema: capital-type, class-position, worker-relationship, ideological-function, serves-capital-type). Amendments ADR-0010 (surveillance state) and ADR-0011 (reproductive rights).

## Section 6: The Contradictions

**Purpose:** highlight internal contradictions. The "both sides" / "against their interests" / "inconsistent position" pattern.

**Auto-rendered:**
- Both-sides flag from `scripts/contradiction-miner.cjs` (same donor funds D and R)
- Voting contradictions (politician voted against stated position)
- Stated-position vs action gap

**Editorial slot:** framing paragraph. Research Claude.

**Can be empty** for profiles with no detected contradiction. In that case the section renders as "No contradictions detected as of [date]" with a link to report one via the corrections form.

## Section 7: Timeline

**Purpose:** chronology of the relationship between this entity and the political system.

**Auto-populated events:**
- FEC cycle contributions (donor profiles)
- Congressional elections won/lost (politician profiles)
- Major bills sponsored + outcomes (politician profiles)
- Contract awards (corporation profiles)
- News events from `data/events.jsonl`
- Executive orders (for presidents and cabinet)

**Editorial slot:** 3-5 timeline entries with editorial significance (e.g. "2017: endorsed Trump while previously funding opposition" with 1-sentence context).

**Format:** reverse-chronological vertical timeline. Most recent first.

## Section 8: Related Figures

**Purpose:** navigation to connected profiles.

**Auto from `data/relationships.jsonl`:** not from frontmatter. Rendered as three columns:
- Funds / Funded by
- Allies (related edges)
- Opposes

**Links:** always to Master Profile or canonical entry, never to a sub-note.

**Editorial slot:** none. This section is data-only.

## Section 9: Sources

**Purpose:** every factual claim traceable.

**Auto-rendered from source registry (`data/sources.jsonl`):**
- Grouped by tier (Tier 1 government + Tier 2 journalism + Tier 3 aggregator + Archived)
- Count per tier
- Every `{{src:ID}}` in the body resolves to a full citation here

**Editorial slot:** none. Automatic.

**Dead URLs:** the source registry status field marks `dead`, `redirected`, `generic_orphan`, `paywall`, `archived`. Dead sources stay in the Archived section with strikethrough. Never silently deleted.

## Frontmatter contract

Every profile that conforms to this template has these frontmatter fields. Missing fields block verified-tier promotion.

### Common (all types)

```yaml
title: "Canonical Name"
type: politician | donor | corporation | state-politician | local-politician | pac | think-tank | lobbying-firm | media-profile
content-readiness: raw | draft | ready | verified
last-updated: YYYY-MM-DD
last-enriched: YYYY-MM-DD
source-tier: 1 | 2 | 3 | 4
parent: null
issues: [list of issues]
```

### Politicians

```yaml
party: "Republican" | "Democrat" | "Independent"
chamber: "Senate" | "House" | "Presidential" | "Cabinet"
state: "Full State Name"
state-abbr: "XX"
district: "NN" (House only)
current-office: "Senator" | "Representative" | etc.
fec-candidate-id: "S4CA00522" | "H8CA45130" | "P80001571"
fec-candidate-id-house: "HXXX" (if they've also run House)
bioguide-id: "A000383"
govtrack-id: 412733
career-total: "$X"
top-donors: [curated top 5-10 for display — rebuild from edges]
```

### Donors

```yaml
sector: "Mega-Donors" | "Defense & Intelligence" | "Energy & Utilities" | etc.
entity-type: "Individual Donor" | "Corporation" | "PAC" | "501(c)(4)" | "Super PAC"
politicians-funded: [curated top 5-10 — rebuild from edges]
issues: [list]
```

### Corporations

```yaml
sector: as above
entity-type: "Corporation"
lei: (LEI identifier if available)
naics-code: (NAICS classification)
total-revenue: "$X"
lobbying-spend: "$X"
federal-contracts: [count, total amount]
ein: (Employer ID)
```

### Relationship cache fields (READ-ONLY — rebuilt from canonical store)

Never hand-edit these. Always updated by `rebuild-relationship-caches.cjs`:

```yaml
related: "[[...]] · [[...]]"
donors: "[[...]] · [[...]]"
top-donors: [...]
politicians-funded: "[[...]]"
opposes: "[[...]]"
stories: "[[...]]"
```

### Internal / roadmap markers (hidden from public render)

```yaml
internal-notes: "[JANITOR ...] free-text"
known-gaps: "description of what's missing"
needs-reenrichment: true | false
reenrich-reason: "why"
editor-vouched: true | false (hallucination-catcher exemption)
featured-date: YYYY-MM-DD (Weekly Spotlight)
```

## Validation

Before promoting to verified:

1. Run `scripts/profile-template-validator.cjs <profile-path>` — checks section presence + order
2. Run `scripts/publication-readiness-check.cjs <profile-path>` — 8-point publication gate
3. Review placeholder markers: all `(URL NEEDED)` / `(UNVERIFIED)` / `(NEEDS REVIEW)` cleared or consciously preserved as roadmap
4. Confirm data freshness: `last-enriched` is less than 90 days old
5. David signs off on the 3 editorial sections (Who They Are, Class Analysis, The Contradictions framing)
6. Promote `content-readiness` to `verified`

## Evolution

This template is load-bearing. Changes require an ADR. Open proposals should go through:
1. Draft the change as an ADR in `content/Decisions/NNNN-profile-template-amendment.md`
2. Update `profile-template-validator.cjs` to match
3. Update existing verified profiles (or grandfather them with a migration script)

Never silently drift the template.
